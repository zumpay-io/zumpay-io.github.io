Getting Started

To Submit A Payment Request

You will need to create a JSON object which consists of the following 3 items:

  1. Amount: This is the amount you would like to receive.
  2. Address: This is your wallet address where you would like to receive the payment.
  3. Callback: This is a callback function link to be executed upon payment submission.
{ "amount": 100000000, "address": "Zum1yfQFeUajU77wyHMvBiMdiQdKgj9qfEc3oiEhHfizSu8wZ6K8ZoWQQY57qNmDC48yg6UAHzyTugB9tMohMnUEArSCdojkxH6", "callback": "https://yourwebapp.com/process" }

Note: ZumCoin uses eight decimals places, so 1.00000000 ZUM becomes 100000000 in atomic units.

Submit The Request To ZumPay™

Post the request to https://api.zumpay.io/v1/new using your favorite programming language. A new wallet will be created (dedicated view and spend keys) just for your request.

Process the Response

If all goes well, you'll receive back a response as such.

{ "sendToAddress": "Zum1yfbF5775QDCRdyK7V4CPvHSF5zkDuBe32Y6FLnKRJdqwp58P9ar3icXHnLPDEAKYaWmnBtmpWBUhPFVC6NFzPZmhb3UdWbE", "atomicAmount": 100000000, "amount": 1, "userDefined": {}, "startHeight": 1109310, "endHeight": 1109370, "confirmations": 60, "qrCode": "https://chart.googleapis.com/chart?cht=qr&chs=256x256&chl=zumcoin://Zum1yfbF5775QDCRdyK7V4CPvHSF5zkDuBe32Y6FLnKRJdqwp58P9ar3icXHnLPDEAKYaWmnBtmpWBUhPFVC6NFzPZmhb3UdWbE?amount=100000000" }

Display the sendToAddress to the user and prompt them to make payment to that address in the proper amount as requested.

Note: The Payment requests are only valid for approximately 60 minutes (exactly 30 blocks). So if your user doesn't complete the payment within the time limit, the request is cancelled.

But if the user attempts to pay you after the expiration of the initial request, we will initiate a fresh request and wait for the user to submit the payment.

Once we receive the funds, or if something goes wrong, we'll send a message to the callback URL you specified. As soon as your user's payment has enough block confirmations we will immediately forward the funds to the address you specified in your request and send a message to the callback URL you provided with the transaction hash containing your funds (minus the network transaction fee of 0.10 ZUM).

The documentation regarding the information supplied in the callbacks is available in the Public API Interface documentation.
